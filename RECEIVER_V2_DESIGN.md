# Receiver v2 — 設計書

## 概要

**目的**: 既存送信機（`nab` バイナリ）とのパケット互換を維持しながら、クロックドリフトによる音切れを根本的に解消する新しい受信機。

**送信機は一切変更しない。受信機のみ新規追加。**

---

## 問題の根本原因

送信側 Windows と受信側 Mac は独立したクロックで動いている。

```
Windows: 48000.0 Hz（実際は 48000 ± 数十ppm）
Mac:     48000.0 Hz（実際は 48000 ± 数十ppm）
```

差は典型的に **±50〜100 ppm**。
48kHz × 100ppm = **4.8サンプル/秒**のドリフト。
ジッターバッファを増やしても解決しない（バッファが徐々に空になる or 溢れる → ブツッ）。

---

## 解決策：アダプティブSRC（Adaptive Sample Rate Conversion）

バッファの残量を長期的に監視し、リサンプリングレートを動的に微調整することでドリフトを吸収する。

```
バッファ残量の長期トレンド
  ↓ 増えてる → 送信側が速い → SRCレシオを上げて再生を少し速める
  ↓ 減ってる → 送信側が遅い → SRCレシオを下げて再生を少し遅くする

調整量: ±0.005%〜±0.5%（人間の耳では絶対に聞き取れない）
```

## バイナリ構成

| バイナリ | ファイル | 役割 |
|----------|----------|------|
| `nab` | `src/main.rs` | 既存（送受信・変更なし） |
| `nab-recv` | `src/bin/receiver.rs` | **新規** 受信機v2 |

ビルド：
```bash
# 既存送信機（変更なし）
cargo build --release --bin nab

# 新受信機
cargo build --release --bin nab-recv
```

## データフロー

```
[送信側PC（Windows / Pyramix等）]
      ↓ UDP 1028バイト/パケット
      ↓ ポート 8000

[受信機 nab-recv]

  ┌─────────────────────────────────────────────┐
  │ UDP受信スレッド                              │
  │  - ソケット受信（100msタイムアウト）          │
  │  - シーケンス番号検証・パケットロス補完       │
  │  - raw_buf（リングバッファ）へプッシュ        │
  └───────────────┬─────────────────────────────┘
                  │ f32サンプル（インターリーブ）
                  ▼
  ┌─────────────────────────────────────────────┐
  │ SRCスレッド（メイン処理）                    │
  │  1. raw_bufから512フレーム読み出し            │
  │  2. インターリーブ → プレーナー変換           │
  │     [L,R,L,R...] → [LLLL...], [RRRR...]     │
  │  3. rubato SincFixedIn でリサンプリング       │
  │  4. プレーナー → インターリーブ変換           │
  │  5. play_bufへプッシュ                       │
  │  6. 200ms毎にドリフト計算・レシオ更新        │
  └───────────────┬─────────────────────────────┘
                  │ f32サンプル（リサンプリング済み）
                  ▼
  ┌─────────────────────────────────────────────┐
  │ CPAL出力コールバック                         │
  │  - play_bufからサンプルを取り出し再生        │
  │  - バッファ不足時 → 無音 + リバッファリング  │
  └─────────────────────────────────────────────┘
```

---

## ドリフト補正アルゴリズム

**比例制御（Pコントローラ）**:

```
target_fill = jitter_ms × sample_rate × channels / 1000
error       = raw_buf.len() - target_fill
adjustment  = error × P_GAIN            （P_GAIN = 3×10⁻⁷）
new_ratio   = clamp(1.0 + adjustment, 0.995, 1.005)

resampler.set_resample_ratio(new_ratio, ramp=true)
```

- `ramp=true` でレシオを急変させずスムーズに補間（クリックノイズ防止）
- 調整範囲: **±5000 ppm**（最大±0.5%）
- 実際のドリフトは±100 ppm程度なので余裕十分

## パケットフォーマット（送信機と同一・変更なし）

```
[0..3]  シーケンス番号 u32 little-endian
[4..1027] サンプルデータ 128フレーム × 2ch × f32 = 1024バイト
合計: 1028バイト
```

## 起動オプション

```bash
# デフォルト（ポート8000、48kHz）
./nab-recv

# ポート・サンプルレート指定
./nab-recv [sample_rate] [port]

# 例
./nab-recv 96000 8000
./nab-recv 44100 9000
```

|----------|--------|------|

## キー操作

| キー | 動作 |
|------|------|
| `+` / `=` | ジッターターゲット +50ms（最大2000ms）|
| `-` | ジッターターゲット -50ms（最小50ms）|
| `q` × 2 | 終了（1秒以内に2回） |

---

## 既存コードとの違い

| 項目 | 既存受信機（nab） | 新受信機（nab-recv） |
|------|-----------------|-------------------|
| ドリフト対策 | なし（ジッターバッファのみ）| **アダプティブSRC** |
| 音切れ原因 | クロックドリフトで定期的に発生 | **SRCが吸収するので発生しない** |
| リサンプリング | なし | rubato SincFixedIn |
| スレッド数 | 2（UDP + CPAL） | 3（UDP + SRC + CPAL） |
| 音質への影響 | — | 調整量が微小なので実質劣化なし |
